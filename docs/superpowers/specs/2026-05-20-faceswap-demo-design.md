# FaceSwap デモアプリ 設計ドキュメント

- **日付**: 2026-05-20
- **作成者**: r-zong@smc-tech.com
- **ステータス**: ドラフト(エンジニアリングレビュー済)

## 目的

ユーザーの顔をテンプレート動画(basketball.mp4 等)に合成し、「自分が神プレイをしている動画」を生成・配布するブラウザ完結のデモアプリ。

- **用途**: デモ・プロトタイプとして見せる
- **コスト**: 完全無料
- **処理時間**: 1分以内
- **品質**: 不自然じゃない範囲、「神プレイ感」優先

## 主要要件

| 項目 | 要件 |
|---|---|
| 形態 | Web アプリ(ブラウザ完結) |
| ホスト | Windows ノートPC 単体 |
| 顔取得 | 本番=ブラウザカメラ、stg=画像アップロード |
| テンプレ | basketball.mp4 固定(将来拡張可能な構造) |
| 結果配布 | デモ画面でその場で再生 + QRコードでスマホDL |
| 遠隔アクセス | 別ネットワーク・別端末からも閲覧可能 |
| 待ち時間UX | バスケテーマのミニゲーム(姿勢推定) |

## アーキテクチャ

```
┌─────────────────────────────────────────────────────┐
│  ブラウザ(誰でもどこからでも)                     │
│  ① ランディング: テンプレ動画ループ背景 + START      │
│  ② 顔キャプチャ: getUserMedia or 画像アップロード    │
│  ③ ミニゲーム: MediaPipe Pose で姿勢推定+バスケ動作  │
│  ④ 結果画面: 動画再生 + QRコード(DL専用URL)         │
└─────────────────────────────────────────────────────┘
                       ↓ HTTPS
┌─────────────────────────────────────────────────────┐
│  Cloudflare Tunnel (cloudflared quick tunnel)       │
│  https://xxxxx.trycloudflare.com                    │
└─────────────────────────────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────────┐
│  Windows ノートPC                                    │
│  ┌────────────────────────────────────────────┐    │
│  │ FastAPI (uvicorn)                          │    │
│  │  GET  /              → index.html          │    │
│  │  POST /api/swap      → ジョブ開始           │    │
│  │  GET  /api/job/:id   → SSE進捗             │    │
│  │  GET  /api/result/:id.mp4 → ストリーミング │    │
│  │  GET  /api/download/:id.mp4 → DL(添付)     │    │
│  └────────────────────────────────────────────┘    │
│           ↓                                          │
│  ┌────────────────────────────────────────────┐    │
│  │ FaceSwap Engine (Python)                   │    │
│  │  InsightFace + inswapper_128.onnx          │    │

│  │  (CUDA → DirectML → OpenVINO → CPU)        │    │

│  └────────────────────────────────────────────┘    │
│           ↓                                          │

└─────────────────────────────────────────────────────┘
```

## UIフロー

```
① ランディング画面
   背景: basketball.mp4 ループ(ミュート、autoplay、loop)
   オーバーレイ: 「📸 あなたが主役になる START を押して」
                          ↓ クリック
② 顔キャプチャ画面
   本番モード: getUserMedia → カメラプレビュー → シャッター
   stgモード(?mode=stg): <input type=file> でアップロード
   撮影後: 確認画面(再撮影 or OK)
                          ↓ OK
③ ミニゲーム + 処理中画面
   背景: テンプレ動画ループ(音量小)
   手前: カメラ映像 + MediaPipe Pose スケルトン描画
   ミッション: ドリブル動作・シュート動作カウント
   進捗バー: ゲーム進捗 + サーバー進捗の合算表示
   解禁条件: 30秒経過 AND バックエンド完了 → ④へ自動遷移
                          ↓
④ 結果画面
   全画面で結果動画再生(右下に自分の顔ワイプ)
   右側に QRコード(DL用URL)
   「もう一回」ボタン → ①へ
```

## データフロー

```
T=0    [START押下] → カメラ起動
T=5s   [シャッター] → 顔キャプチャ
T=8s   [OK押下]
        │ ① POST /api/swap (multipart: face.jpg)
        │   サーバー: job_id 発行、非同期処理開始
        │   即座に { job_id } を返す
        │ ② フロント: ミニゲーム画面へ遷移
        │   SSE接続 /api/job/{job_id}
        │   MediaPipe Pose 開始
        │
T=8〜  [並列実行]
        ├─ サーバー側:
        │   1. アップロード顔の検出 + embedding 抽出
        │   2. テンプレフレーム(プリ抽出済121枚)読み込み
        │   3. バッチ4枚単位でinswapper実行(SSE進捗送信:5フレ毎)
        │   4. ffmpeg 1コマンドで動画再構築:
        │      - 元音声トラックを保持
        │      - 右下にユーザー顔静止画をワイプ overlay
        │   5. result.mp4 完成 → SSE "done" イベント
        │
        └─ クライアント側:
            1. ミニゲーム進行
            2. 進捗表示
            3. (30秒経過 AND サーバー完了) で結果解禁
            4. 未達なら「もう少し!」と煽る or 演出継続
        │
T=??s  [両方完了] → 結果画面へ遷移
        <video src="/api/result/{job_id}.mp4"> で再生
        QR には /api/download/{job_id}.mp4 を埋め込み(Content-Disposition: attachment)
        QRスキャン → スマホで即DL

T=完了+10分 [サーバー側自動]
        jobs/{job_id}/ ディレクトリを自動削除
```

## 技術スタック

### フロントエンド
- **形式**: シングルHTML + Vanilla JS(ビルド不要、Windows依存最小)
- **カメラ**: `navigator.mediaDevices.getUserMedia` + `<canvas>`
- **姿勢推定**: MediaPipe Pose(JS版、CDN読み込み)
- **QRコード**: `qrcode-svg` または `qrcode.js`(CDN)
- **進捗**: Server-Sent Events (SSE)、ポーリングフォールバック
- **スタイル**: Tailwind CDN または素CSS

### バックエンド
- **言語**: Python 3.11+
- **Web**: FastAPI + uvicorn
- **パッケージ管理**: uv または venv + pip
- **ジョブ管理**: メモリ内 dict + asyncio.Task(MVP方針、YAGNI)

### 推論パイプライン
- **顔検出**: InsightFace `buffalo_l`(必要に応じて軽量版 `buffalo_s`)
- **顔スワップ**: `inswapper_128.onnx`
  - 注: 公式配布停止、ミラー取得。将来 SimSwap へ移行候補(TODOS参照)
- **ランタイム**: ONNX Runtime
- **EP優先順位**: CUDAExecutionProvider → DmlExecutionProvider → OpenVINOExecutionProvider → CPUExecutionProvider
- **起動時最適化**:
  - 推論セッションをプリロード(常駐)
  - テンプレフレームをプリ抽出して `templates/basketball/frame_000.jpg` 形式で保存
  - 顔検出結果をテンプレ用に1度だけ実行してキャッシュ

### 動画処理
- **分解/合成**: ffmpeg(subprocess経由)
- **音声処理**: 元動画から音声トラック分離→再合成時にマージ
- **ワイプ合成**: ffmpeg `overlay` フィルタ
  - 配置: 右下、`overlay=W-w-20:H-h-20`
  - サイズ: 入力動画の 1/4 幅程度
- **結果フォーマット**: MP4 (H.264 + AAC)、モバイル互換

### 公開・配信
- **トンネル**: cloudflared quick tunnel(現MVP)
  - 将来: Named Tunnel への切替(TODOS参照)
- **起動**: `start.bat`(Windows)で uvicorn + cloudflared を一括起動

## 機能リスト

### 採用
- 音声保持(テンプレ動画の音声を結果動画にマージ)
- 進捗バー(SSEベース、5フレーム毎更新)
- 顔検出失敗時のエラー表示と再撮影フロー
- ジョブ自動削除(完了後10分)
- stgモード(URL `?mode=stg` で画像アップロード可)
- ワイプ合成(結果動画右下にユーザー顔の静止画)
- QRコードによるスマホDL(`Content-Disposition: attachment`)
- ランディング背景のテンプレ動画ループ(変身ビフォー演出)
- 待ち時間ミニゲーム(MediaPipe Pose、ドリブル+シュート)
- 30秒経過 AND バックエンド完了 で結果自動解禁
- 「もう一回」ボタン

### 不採用(MVP範囲外)
- Before/After 比較表示(合成感が出るため意図的に除外)
- 顔の高品質化(GFPGAN等、処理時間優先)
- 複数テンプレート切替(将来拡張可能な構造のみ用意)
- 複数人スワップ
- シェアボタン(SNS)
- ウォーターマーク
- 過去生成ギャラリー
- 多言語(JP固定)
- ユーザーアカウント
- リアルタイム配信

## エラー処理・エッジケース

| シナリオ | 対応 |
|---|---|
| カメラ拒否 | 「カメラを許可してください、または stg モードへ」 |
| 顔が検出できない | 「顔がはっきり映っていません、撮り直してください」 + 再撮影 |
| 複数の顔が検出 | 一番大きい(=メイン)顔を自動選択 |
| テンプレに顔がない | サーバー起動時にテンプレ検証、欠落なら起動失敗 |
| ONNX EP失敗 | EP優先順位で次にフォールバック、最後はCPU |
| 処理タイムアウト(120秒超) | ジョブ中断、「混雑しています」表示 |
| MediaPipeロード失敗 | ゲームスキップ → 通常の待ち画面に縮退 |
| SSE切断 | 自動再接続、ダメなら polling |
| 同時複数ジョブ | キュー化(直列処理1つずつ) |
| モデルDL失敗(初回起動) | エラー表示 + リトライ案内 |
| ゲーム100%未到達 | 30秒経過で自動解禁 |

## 性能最適化(設計に含めるもの)

1. **モデル/セッションのプリロード**: 起動時にONNXセッション常駐
2. **テンプレフレームのプリ抽出**: 起動時に1回、ディスクキャッシュ
3. **テンプレの顔検出キャッシュ**: テンプレ側は1回だけ検出
4. **バッチ推論**: 4フレーム単位でinswapper呼び出し → CPU/iGPUで約2倍速
5. **ffmpeg統合**: 分解+合成+音声+ワイプを **1コマンド** にまとめる
6. **SSE間引き**: 進捗イベントは5フレームに1回だけ送信(トンネル経由のlatency対策)

## テスト計画

### ユニット/統合(pytest + httpx)
1. ゴールデンテスト: basketball.mp4 + 固定顔写真 → 結果動画のSSIM > 0.95
2. 顔ゼロ枚: 単色画像入力 → 400 + 「顔が検出できません」
3. 音声保持: 結果動画の音声ストリーム長・コーデック検証
4. EP fallback: CUDA を強制失敗 → DML → CPU の順で試行確認
5. タイムアウト: 120秒超のジョブが中断される
6. クリーンアップ: 10分経過後 jobs/ ディレクトリが空
7. ワイプ位置: 結果動画の右下20pxにワイプが正しい位置
8. E2E: 画像アップロード(stg) → ジョブ完了 → 結果取得まで
9. MediaPipe フォールバック: モデルロード失敗時にゲームスキップ
10. 回帰テスト: 顔検出キャッシュ最適化を入れる前後で出力品質劣化なし

### 手動テスト

- iOS Safari の自動再生制約挙動

## 失敗モードと対策

| 失敗モード | 対策 |
|---|---|
| inswapper モデル入手経路が消える | TODOS化、SimSwap移行を事前検証 |
| Cloudflare Tunnel URL変動でデモ崩壊 | 起動時にQRをログ出力、必要なら Named Tunnel化 |
| ノートPC性能不足で処理1分超過 | 解像度ダウンスケール(360x640)+ フレーム間引きで強制1分以内 |
| カメラ二重使用の競合 | キャプチャ後 `track.stop()` で明示解放、MediaPipe前に再acquire |
| 顔ワイプの境界線が荒い | ワイプ画像を円形マスク + ぼかしで馴染ませる |
